## 2024-05-24 - File and Command Caching for Repetitive Context Building
**Learning:** The agent loop continuously queries skill availability and reads context files repeatedly for every LLM chat interaction, creating significant disk I/O bottlenecks and latency during conversational tasks because Python's built-in file open and `shutil.which` are synchronous blocking operations.
**Action:** When working on systems that require periodic or repetitive context loading across agent iterations, always implement an in-memory caching mechanism with `os.path.getmtime` for files and straightforward dictionary checks for static configurations like available CLI binaries (`shutil.which`).
