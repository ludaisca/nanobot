## 2026-02-12 - [Async Memory Consolidation]
**Learning:** Moving heavy LLM operations (like memory consolidation) to background tasks requires careful handling of shared mutable state (sessions). Using an offset-based approach for trimming (`messages[n_removed:]`) is safer than negative slicing (`messages[-keep:]`) when the list can grow concurrently. Also, always track background tasks in a set to prevent garbage collection.
**Action:** When optimizing long-running operations in async loops, always ensure state consistency and task lifecycle management.

## 2025-04-07 - [Optimizing Python String Operations]
**Learning:** When optimizing Python string operations like case conversions (`camel_to_snake`, `snake_to_camel`), pre-compiled regular expressions (like `re.compile(r'(?<!^)(?=[A-Z])')`) can be significantly faster than character-by-character loops. Additionally, list comprehensions are faster than generator expressions inside `"".join()`. Adding fast-paths for already-formatted strings (e.g., checking `islower()` or `_ not in name`) drastically reduces execution time.
**Action:** Always consider fast-paths and pre-compiled regex for frequent string operations, and prefer list comprehensions over generator expressions when passing arguments to `join()`.
